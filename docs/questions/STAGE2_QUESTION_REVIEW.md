## 3. 发布阻塞复审记录

### B1：结构化题答案随练习详情泄露——已关闭

当前 `PracticeQuestion` 已移除：

- `solutionCode`、`hiddenTests` 和 `testHarnessCode`；
- 选择题的 `correctChoiceIds`；
- 排序题的 `correctCodeOrder`；
- output、multipleChoice、codeOrder 三类题公开示例中的 `expectedOutput`。

受影响题目：

`ch02-q001`、`ch03-q001`、`ch04-q001`、`ch06-q001`、`ch09-q001`、`ch10-q001`、`ch11-q001`、`ch12-q001`、`ch14-q003`。

Provider 回归测试逐项验证上述字段不会进入公共详情，受影响的 9 道结构化题不再泄露答案。

### B2：`exact` 与 stdout 末尾换行协议未落定——已关闭

规范和比较器现已统一：`exact` 只把 CRLF/CR 规范化为 LF，其他空格及终止换行必须完全一致。比较器测试确认 `a\n` 与 `a` 不相等，末尾空格也不会被忽略。

题目源数据已直接修订：stdout 执行型 `exact` 测试的 `expectedOutput` 显式包含终止 `\n`；output、multipleChoice、codeOrder 继续按结构化答案处理。`factory.ts` 保持纯构造，不再静默转换比较模式。validator 会拒绝缺少终止换行的 stdout `exact` 数据。

复审执行 `validate:questions`、全量测试、lint 和 build 均通过，B2 关闭。

## 6. 非阻塞改进建议

- 固定答案的 output、multipleChoice、codeOrder 题目前用重复的 visible/hidden 数据满足统一 Schema，隐藏用例没有新增边界价值。后续应为结构化题建立专用答案与判定 Schema。
- 在自动校验中加入“公开详情不得含答案字段”的快照测试。
- 加入典型错误答案回归，例如只按 `% 4` 判断闰年、使用集合破坏首次出现顺序、用类属性保存可变列表等，形成规范要求的错误拦截证据。
- 静态校验可继续增加题型互斥字段检查，例如非选择题禁止出现 `correctChoiceIds`，非排序题禁止出现 `correctCodeOrder`。
